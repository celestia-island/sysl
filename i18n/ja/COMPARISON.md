## 詳細分析

### SySL と MIT

**MIT を選ぶ場合**: コードが人間によって書かれ、最大限のシンプルさと許容性を求め、特許保護やAI特有の条件を必要としない場合。

**SySL を選ぶ場合**: コードが大幅にAI生成されており、ユーザーにそのことを知らせたい、そして著作権が適用されなくてもライセンスが機能することを望む場合。

**主な違い**: MIT は純粋な著作権ライセンスです。著作権がAI生成コードを対象としない場合、MIT は法的枠組みを提供しません。SySL は著作権ライセンスと契約の両方として機能します。

### SySL と Apache 2.0

**Apache 2.0 を選ぶ場合**: コードが人間によって書かれ、特許保護を備えた許容的ライセンスを求め、エンタープライズ環境で運用する場合。

**SySL を選ぶ場合**: MIT との比較と同様に加えて：SySL は Apache 2.0 と互換性を持つように設計されています。SySL でライセンスされたコードを Apache-2.0 でライセンスされたコードと組み合わせ、AI開示が保持されることを条件に Apache 2.0 の下で配布できます。

**主な違い**: Apache 2.0 は特許許諾を持ちます。SySL も同等の特許許諾を持ちます。決定的な違いは、Apache 2.0 は著作権を前提とし、契約による代替を提供しないことです。SySL の特許許諾は著作権から独立して機能します。

### SySL と GPLv3

**GPLv3 を選ぶ場合**: 強力なコピーレフトを求める場合——あなたのコードを改変して再配布する者は、その改変物も GPL の下で配布しなければなりません。

**SySL を選ぶ場合**: 透明性を求めるが、ソース共有の強制は求めない場合。SySL の「合成的コピーレフト」はAI開示が保持されることのみを要求し、プロジェクト全体をオープンソース化することを求めません。

**主な違い**: GPL のコピーレフトはコードを開かれたままにすることに関するものです。SySL のコピーレフトはAI開示の連鎖を維持することに関するものです。SySL コードを取得し、プロプライエタリな改変を加え、プロプライエタリライセンスの下で配布できます——AI開示が残る限り。

### SySL と BUSL 1.1

**BUSL 1.1 を選ぶ場合**: 商業的製品を競合他社によるクローンから保護し、コードが最終的にオープンソースに変換されることを望む場合。

**SySL を選ぶ場合**: 透明性の規範とともに最大限の普及を求める場合。非競合期間も自動変換もありません。

**主な違い**: BUSL は防御的ライセンス（ビジネスを保護）です。SySL は透明性ライセンス（社会的規範を確立）です。両者は異なる目的を果たし、デュアルライセンスを通じて共存できます（SySL §11.3）。両方を使用するAI生成プロジェクトでは、BUSL の変更ライセンスを Apache-2.0 ではなく SySL に設定すべきです——さもなければ、非競合期間が終了した時点で透明性の規範が消滅します。

## 互換性マトリクス

ライセンス A の下のコードをライセンス B の下のコードと組み合わせられますか？

| ライセンス A ↓ / ライセンス B → | MIT | Apache 2.0 | GPLv3 | SySL 1.0 | BUSL 1.1 |
|:---|:---:|:---:|:---:|:---:|:---:|
| **MIT** | ✓ | ✓ | ✓ | ✓ | ✓ |
| **Apache 2.0** | ✓ | ✓ | ✓（v3 のみ） | ✓ | 条件付 |
| **GPLv3** | ✓ | ✓（v3 のみ） | ✓ | 検討中 | ✗ |
| **SySL 1.0** | ✓ | ✓ | 検討中 | ✓ | 条件付 |
| **BUSL 1.1** | ✓ | 条件付 | ✗ | 条件付 | ✗ |

*✓ = 互換性が確認済み。✗ = 非互換性が確認済み。「検討中」= 未分析。「条件付」= 特定の条件に依存。*

## どのライセンスを使うべきか？

```
Is your code substantially AI-generated?
  ├── Yes → Are you comfortable with users not knowing?
  │         ├── Yes → MIT or Apache 2.0
  │         └── No  → SySL
  └── No  → Do you need patent protection?
              ├── Yes → Apache 2.0
              └── No  → MIT
```

AI生成コードに特化して：

```
Do you want derivatives to keep AI disclosure?
  ├── Yes → SySL
  └── No  → MIT or Apache 2.0 (acknowledging legal uncertainty)
```
