                             PRÉAMBULE

La Licence de Source Synthétique (SySL) est conçue pour les logiciels
dont le code source est substantiellement ou entièrement généré par des
modèles d'intelligence artificielle. Les licences open source
traditionnelles présupposent un auteur humain titulaire de droits
d'auteur sur l'œuvre. La SySL reconnaît que le code généré par l'IA
occupe une position juridique inédite — une position dans laquelle la
protection par le droit d'auteur peut être incertaine, absente ou
contestée — et fournit un cadre qui fonctionne indépendamment de cette
incertitude.

La SySL établit trois normes nouvelles pour l'ère de l'IA :

  1. TRANSPARENCE. Chaque utilisateur et chaque œuvre dérivée doit
     divulguer que le code a été généré par IA et quels modèles ont
     été utilisés.

  2. SÉCURITÉ JURIDIQUE. La licence fonctionne comme un contrat
     contraignant indépendant du droit d'auteur, de sorte que
     l'incertitude quant au statut du droit d'auteur n'engendre pas
     d'incertitude quant à l'utilisation autorisée.

  3. RECONNAISSANCE DES RISQUES. Le code généré par l'IA présente
     des modes de défaillance distincts de ceux du code écrit par
     un humain. Les utilisateurs l'acceptent explicitement.

Ces normes ne constituent pas des restrictions à la liberté. Elles sont
le fondement d'un nouveau contrat social : nous pouvons construire des
logiciels avec l'IA, mais nous devons le dire.

                 Comment Appliquer la SySL à Votre Logiciel

Pour appliquer la Licence de Source Synthétique à votre œuvre, joignez
l'avis type suivant à chaque fichier source. L'avis doit être placé dans
la syntaxe de commentaire appropriée au format du fichier.

   Copyright (c) [année] [votre nom ou organisation]

   Sous licence Synthetic Source License, Version 1.0
   (la « Licence ») ; vous ne pouvez utiliser ce fichier qu'en
   conformité avec la Licence. Vous pouvez obtenir une copie de la
   Licence à l'adresse suivante :

       https://github.com/celestia-island/sysl

   Sauf disposition contraire de la loi applicable ou convention
   écrite, le logiciel distribué sous la Licence est fourni
   « EN L'ÉTAT », SANS GARANTIES NI CONDITIONS D'AUCUNE SORTE,
   expresses ou implicites. Voir la Licence pour le libellé
   spécifique régissant les autorisations et les limitations.

   DIVULGATION DES MODÈLES D'IA : [Décrivez quels modèles d'IA
   ont été utilisés pour générer ce logiciel. Voir la Section 2
   de la Licence.]

Créez en outre un fichier nommé AI_DISCLOSURE (ou AI_DISCLOSURE.md)
à la racine de votre dépôt décrivant les modèles d'IA utilisés, le
nombre approximatif de jetons consommés et la proportion de contenu
généré par IA. Voir la Section 2.2 de la Licence pour l'exigence
complète.

Pour placer votre logiciel sous licence « SySL-1.0 ou toute version
ultérieure », vous pouvez inclure la mention « ou (à votre choix)
toute version ultérieure » dans l'avis type ci-dessus. Si vous
n'incluez pas cette mention, votre logiciel n'est concédé que sous
la Version 1.0.
