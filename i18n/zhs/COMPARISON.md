# 许可证对比：SySL 与传统许可证

## 一览

| | SySL 1.0 | MIT | Apache 2.0 | GPLv3 | BUSL 1.1 |
|---|:---:|:---:|:---:|:---:|:---:|
| **创建年份** | 2026 | 1988 | 2004 | 2007 | 2016 |
| **类型** | 合成源码 | 宽松 | 宽松 + 专利 | 强著佐权 | 延迟开源 |
| **需要著作权** | 否（合同后备） | 是 | 是 | 是 | 是 |
| **AI 感知** | 是（核心设计） | 否 | 否 | 否 | 否 |
| **专利授权** | 是（有条件：仅当作者持有专利时） | 否 | 是 | 是 | 否 |
| **允许 AI 训练** | 是（明确，§9.1） | 未规定 | 未规定 | 未规定 | 未规定 |
| **侵权丧失** | 是（§9.2，自动丧失） | 否 | 否 | 否 | 否 |
| **著佐权** | 仅披露 | 否 | 否 | 是（强） | 否 |
| **商业使用** | 是（须披露） | 是 | 是 | 是（须源代码） | 非竞争期 |
| **分发条款** | 保留 AI 披露 | 保留著作权声明 | 保留著作权 + 专利声明 | 以 GPL 分发 | 兼容 FSL |
| **责任免除** | 标准 + AI 特定 | 标准 | 标准 | 标准 | 标准 |
| **OSI 批准** | 否（新类别） | 是 | 是 | 是 | 否 |
| **FSF 自由** | 是（四项自由） | 是 | 是 | 是 | 否 |
| **变更日期** | 无（永久） | 不适用 | 不适用 | 不适用 | 转换为指定许可证 |

## 详细分析

### SySL 与 MIT 对比

**何时选择 MIT**：您的代码由人类编写，您希望最大程度的简洁与宽松，且您不需要专利保护或 AI 特定条款。

**何时选择 SySL**：您的代码主要由 AI 生成，您希望用户了解这一点，并且您希望许可证即使在著作权不适用时也能发挥作用。

**关键区别**：MIT 纯粹是著作权许可证。若著作权不涵盖您 AI 生成的代码，MIT 无法提供法律框架。SySL 则同时作为著作权许可证和合同运行。

### SySL 与 Apache 2.0 对比

**何时选择 Apache 2.0**：您的代码由人类编写，您希望获得带有专利保护的宽松许可证，且您处于企业环境中。

**何时选择 SySL**：与 MIT 对比的情形相同，此外：SySL 被设计为与 Apache 2.0 兼容。您可以将 SySL 许可的代码与 Apache-2.0 许可的代码合并，并以 Apache 2.0 分发，前提是保留 AI 披露。

**关键区别**：Apache 2.0 提供专利授权。SySL 提供等效的专利授权。关键区别在于，Apache 2.0 假定著作权存在且不提供合同后备。SySL 的专利授权独立于著作权运作。

### SySL 与 GPLv3 对比

**何时选择 GPLv3**：您希望强著佐权——任何修改并重新分发您代码的人都必须也以 GPL 分发其修改。

**何时选择 SySL**：您希望透明度，但不希望强制共享源代码。SySL 的"合成式著佐权"仅要求保留 AI 披露，而非将整个项目开源。

**关键区别**：GPL 的著佐权旨在保持代码开放。SySL 的著佐权旨在保持 AI 披露链完整。您可以获取 SySL 代码，添加专有修改，并以专有许可证分发——只要 AI 披露得以保留。

### SySL 与 BUSL 1.1 对比

**何时选择 BUSL 1.1**：您希望保护商业产品免受竞争对手克隆，并最终将代码转换为开源。

**何时选择 SySL**：您希望在透明度规范下获得最大采用率。没有非竞争期，也没有自动转换。

**关键区别**：BUSL 是防御性许可证（保护业务）。SySL 是透明度许可证（建立社会规范）。它们服务于不同目的，可以通过双重许可共存（SySL §11.3）。对于同时使用两者的 AI 生成项目，BUSL 的变更许可证应设为 SySL——而非 Apache-2.0——否则透明度规范将在非竞争期结束后消失。

## 兼容性矩阵

您可以将许可证 A 下的代码与许可证 B 下的代码合并吗？

| 许可证 A ↓ / 许可证 B → | MIT | Apache 2.0 | GPLv3 | SySL 1.0 | BUSL 1.1 |
|:---|:---:|:---:|:---:|:---:|:---:|
| **MIT** | ✓ | ✓ | ✓ | ✓ | ✓ |
| **Apache 2.0** | ✓ | ✓ | ✓（仅 v3） | ✓ | 有条件 |
| **GPLv3** | ✓ | ✓（仅 v3） | ✓ | 审核中 | ✗ |
| **SySL 1.0** | ✓ | ✓ | 审核中 | ✓ | 有条件 |
| **BUSL 1.1** | ✓ | 有条件 | ✗ | 有条件 | ✗ |

*✓ = 已知兼容。✗ = 已知不兼容。"审核中" = 尚未分析。"有条件" = 取决于具体条款。*

## 我应该使用哪个许可证？

```
Is your code substantially AI-generated?
  ├── Yes → Are you comfortable with users not knowing?
  │         ├── Yes → MIT or Apache 2.0
  │         └── No  → SySL
  └── No  → Do you need patent protection?
              ├── Yes → Apache 2.0
              └── No  → MIT
```

针对 AI 生成代码：

```
Do you want derivatives to keep AI disclosure?
  ├── Yes → SySL
  └── No  → MIT or Apache 2.0 (acknowledging legal uncertainty)
```
