# Сравнение лицензий: SySL против традиционных лицензий

## Краткий обзор

| | SySL 1.0 | MIT | Apache 2.0 | GPLv3 | BUSL 1.1 |

| **Год создания** | 2026 | 1988 | 2004 | 2007 | 2016 |
| **Тип** | Синтетический исходный код | Пермиссивная | Пермиссивная + патент | Сильный копилефт | Отложенный открытый исходный код |
| **Требуется авторское право** | Нет (договорная резервная основа) | Да | Да | Да | Да |

| **Предоставление патентных прав** | Да (условно: только если Автор владеет патентами) | Нет | Да | Да | Нет |

## Подробный анализ

### SySL против MIT

**Когда выбрать MIT**: Ваш код написан человеком, вы хотите максимальной простоты и пермиссивности, и вам не нужна патентная защита или условия, специфичные для ИИ.

**Когда выбрать SySL**: Ваш код по большей части создан ИИ, вы хотите, чтобы пользователи об этом знали, и вы хотите, чтобы лицензия работала, даже если авторское право не применимо.

**Ключевое отличие**: MIT — это чисто лицензия на авторское право. Если авторское право не распространяется на код, созданный ИИ, MIT не предоставляет юридической основы. SySL работает и как лицензия на авторское право, и как договор.

### SySL против Apache 2.0

**Когда выбрать Apache 2.0**: Ваш код написан человеком, вы хотите пермиссивную лицензию с патентной защитой, и вы работаете в корпоративной среде.

**Когда выбрать SySL**: То же, что и в сравнении с MIT, плюс: SySL разработана так, чтобы быть совместимой с Apache 2.0. Вы можете объединить код под SySL с кодом под Apache-2.0 и распространять под Apache 2.0 при условии сохранения раскрытия ИИ.

**Ключевое отличие**: В Apache 2.0 есть предоставление патентных прав. В SySL есть эквивалентное предоставление патентных прав. Принципиальное отличие в том, что Apache 2.0 предполагает наличие авторского права и не имеет договорной резервной основы. Предоставление патентных прав по SySL действует независимо от авторского права.

### SySL против GPLv3

**Когда выбрать GPLv3**: Вы хотите сильный копилефт — любой, кто модифицирует и перераспространяет ваш код, должен также распространять свои модификации под GPL.

**Когда выбрать SySL**: Вы хотите прозрачности, но не принуждения к раскрытию исходного кода. «Синтетический копилефт» SySL требует лишь сохранения раскрытия ИИ, а не открытости всего проекта.

**Ключевое отличие**: Копилефт GPL направлен на сохранение кода открытым. Копилефт SySL направлен на сохранение целостности цепочки раскрытия ИИ. Вы можете взять код под SySL, добавить проприетарные модификации и распространять под проприетарной лицензией — при условии сохранения раскрытия ИИ.

### SySL против BUSL 1.1

**Когда выбрать BUSL 1.1**: Вы хотите защитить коммерческий продукт от копирования конкурентами, чтобы код в итоге стал открытым.

**Когда выбрать SySL**: Вы хотите максимального внедрения с нормой прозрачности. Периода неконкуренции и автоматического преобразования нет.

**Ключевое отличие**: BUSL — оборонительная лицензия (защита бизнеса). SySL — лицензия прозрачности (установление социальной нормы). Они служат разным целям и могут сосуществовать через двойное лицензирование (SySL §11.3). Для созданных ИИ проектов, использующих обе лицензии, Замещающая лицензия BUSL должна быть установлена на SySL — а не на Apache-2.0 — иначе норма прозрачности исчезнет по окончании периода неконкуренции.

## Матрица совместимости

Можно ли объединить код под лицензией A с кодом под лицензией B?

| Лицензия A ↓ / Лицензия B → | MIT | Apache 2.0 | GPLv3 | SySL 1.0 | BUSL 1.1 |
|:---|:---:|:---:|:---:|:---:|:---:|
| **MIT** | ✓ | ✓ | ✓ | ✓ | ✓ |
| **Apache 2.0** | ✓ | ✓ | ✓ (только v3) | ✓ | Условно |
| **GPLv3** | ✓ | ✓ (только v3) | ✓ | На рассмотрении | ✗ |
| **SySL 1.0** | ✓ | ✓ | На рассмотрении | ✓ | Условно |
| **BUSL 1.1** | ✓ | Условно | ✗ | Условно | ✗ |

*✓ = заведомо совместимы. ✗ = заведомо несовместимы. «На рассмотрении» = ещё не проанализировано. «Условно» = зависит от конкретных условий.*

## Какую лицензию выбрать?

```
Ваш код по большей части создан ИИ?
  ├── Да → Вас устраивает, что пользователи не будут знать?
  │         ├── Да → MIT или Apache 2.0
  │         └── Нет  → SySL
  └── Нет  → Вам нужна патентная защита?
             ├── Да → Apache 2.0
             └── Нет  → MIT
```

Особенно для кода, созданного ИИ:

```
Вы хотите, чтобы производные произведения сохраняли раскрытие ИИ?
  ├── Да → SySL
  └── Нет  → MIT или Apache 2.0 (учитывая юридическую неопределённость)
```
