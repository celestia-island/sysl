## 상세 분석

### SySL과 MIT

**MIT를 선택해야 하는 경우**: 코드가 사람에 의해 작성되었고, 최대한의 단순함과 허용성을 원하며, 특허 보호나 AI 특유의 조건이 필요하지 않은 경우.

**SySL을 선택해야 하는 경우**: 코드가 상당 부분 AI로 생성되었고, 사용자에게 이를 알리고자 하며, 저작권이 적용되지 않더라도 라이선스가 작동하기를 원하는 경우.

**주요 차이점**: MIT는 순수한 저작권 라이선스입니다. 저작권이 AI 생성 코드를 포괄하지 않는다면, MIT는 법적 프레임워크를 제공하지 않습니다. SySL은 저작권 라이선스와 계약 모두로 작동합니다.

### SySL과 Apache 2.0

**Apache 2.0을 선택해야 하는 경우**: 코드가 사람에 의해 작성되었고, 특허 보호를 갖춘 허용적 라이선스를 원하며, 기업 환경에서 운영하는 경우.

**SySL을 선택해야 하는 경우**: MIT 비교와 동일하며 추가로: SySL은 Apache 2.0과 호환되도록 설계되었습니다. SySL 라이선스 코드를 Apache-2.0 라이선스 코드와 결합하여, AI 공개가 유지되는 조건으로 Apache 2.0으로 배포할 수 있습니다.

**주요 차이점**: Apache 2.0은 특허 허여를 갖습니다. SySL도 동등한 특허 허여를 갖습니다. 결정적 차이는 Apache 2.0이 저작권을 가정하고 계약 대비책을 제공하지 않는다는 것입니다. SySL의 특허 허여는 저작권과 독립적으로 작동합니다.

### SySL과 GPLv3

**GPLv3을 선택해야 하는 경우**: 강력한 카피레프트를 원하는 경우——코드를 수정하여 재배포하는 사람은 자신의 수정본도 GPL로 배포해야 합니다.

**SySL을 선택해야 하는 경우**: 투명성을 원하지만 소스 공유 강제는 원하지 않는 경우. SySL의 "합성 카피레프트"는 AI 공개가 유지될 것만을 요구하며, 전체 프로젝트를 오픈소스화할 것을 요구하지 않습니다.

**주요 차이점**: GPL의 카피레프트는 코드를 개방 상태로 유지하는 것에 관한 것입니다. SySL의 카피레프트는 AI 공개 체인을 온전히 유지하는 것에 관한 것입니다. SySL 코드를 가져와 독점 수정을 추가하고 독점 라이선스로 배포할 수 있습니다——AI 공개가 남아 있는 한.

### SySL과 BUSL 1.1

**BUSL 1.1을 선택해야 하는 경우**: 상업적 제품을 경쟁자의 복제로부터 보호하고, 코드가 결국 오픈소스로 전환되기를 원하는 경우.

**SySL을 선택해야 하는 경우**: 투명성 규범과 함께 최대한의 채택을 원하는 경우. 비경쟁 기간도 자동 전환도 없습니다.

**주요 차이점**: BUSL은 방어적 라이선스(비즈니스 보호)입니다. SySL은 투명성 라이선스(사회적 규범 수립)입니다. 두 라이선스는 서로 다른 목적을 수행하며 이중 라이선싱을 통해 공존할 수 있습니다(SySL §11.3). 둘 다 사용하는 AI 생성 프로젝트에서는 BUSL의 변경 라이선스를 Apache-2.0이 아닌 SySL로 설정해야 합니다——그렇지 않으면 비경쟁 기간이 종료될 때 투명성 규범이 사라집니다.

## 호환성 매트릭스

라이선스 A의 코드를 라이선스 B의 코드와 결합할 수 있나요?

| 라이선스 A ↓ / 라이선스 B → | MIT | Apache 2.0 | GPLv3 | SySL 1.0 | BUSL 1.1 |
|:---|:---:|:---:|:---:|:---:|:---:|
| **MIT** | ✓ | ✓ | ✓ | ✓ | ✓ |
| **Apache 2.0** | ✓ | ✓ | ✓ (v3만) | ✓ | 조건부 |
| **GPLv3** | ✓ | ✓ (v3만) | ✓ | 검토 중 | ✗ |
| **SySL 1.0** | ✓ | ✓ | 검토 중 | ✓ | 조건부 |
| **BUSL 1.1** | ✓ | 조건부 | ✗ | 조건부 | ✗ |

*✓ = 호환됨이 확인됨. ✗ = 비호환됨이 확인됨. "검토 중" = 아직 분석되지 않음. "조건부" = 특정 조건에 따름.*

## 어떤 라이선스를 사용해야 하나요?

```
Is your code substantially AI-generated?
  ├── Yes → Are you comfortable with users not knowing?
  │         ├── Yes → MIT or Apache 2.0
  │         └── No  → SySL
  └── No  → Do you need patent protection?
              ├── Yes → Apache 2.0
              └── No  → MIT
```

AI 생성 코드에 특화하여:

```
Do you want derivatives to keep AI disclosure?
  ├── Yes → SySL
  └── No  → MIT or Apache 2.0 (acknowledging legal uncertainty)
```
